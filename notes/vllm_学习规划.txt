# vLLM 源码学习规划

基于对 vLLM 项目结构的详细分析，制定一份由浅入深、模块化的学习规划。适合有一定 Python 和 LLM 推理基础的开发者，按"理解系统全貌 → 深入核心模块 → 掌握优化技术 → 探索高级特性"的路线展开。

================================================================================
第一阶段：环境搭建与全局概览（1-2 天）
================================================================================

目标：能跑起来 vLLM，理解项目的整体架构和设计哲学。

1.1 阅读关键文档
  - README.md — 项目介绍
  - docs/design/arch_overview.md — 核心必读，理解 V1 多进程架构（API Server →
   Engine Core → GPU Worker），进程数量关系
  - docs/design/huggingface_integration.md — 理解 vLLM 如何加载 HuggingFace 模型
  - CONTRIBUTING.md + AGENTS.md — 了解贡献规范和开发环境搭建（uv、pre-commit）

1.2 环境搭建与调试
  uv venv --python 3.12 && source .venv/bin/activate
  VLLM_USE_PRECOMPILED=1 uv pip install -e . --torch-backend=auto
  - 跑通 examples/basic/ 中的示例脚本
  - 用 vllm serve <model> 启动服务并调用 /v1/completions

1.3 理解配置体系
  - vllm/config/vllm.py — VllmConfig 是全局配置对象，贯穿所有组件
  - vllm/config/model.py、vllm/config/cache.py、vllm/config/parallel.py 等 — 各子配置

1.4 理解入口点
  - vllm/entrypoints/llm.py — LLM 类，离线推理入口
  - vllm/entrypoints/openai/api_server.py — OpenAI 兼容 API 服务器
  - vllm/entrypoints/cli/main.py — CLI 入口

================================================================================
第二阶段：V1 Engine 核心链路（3-5 天）
================================================================================

目标：深入理解 V1 引擎如何接收请求、调度、执行模型并返回结果。

2.1 Engine Core（引擎核心）
  - vllm/v1/engine/core.py — 核心主循环：接收请求 → 调度 → 执行 → 返回结果
  - vllm/v1/engine/core_client.py — 客户端侧通信（ZMQ socket）
  - vllm/v1/engine/llm_engine.py — 高层 LLMEngine 封装

2.2 Scheduler（调度器）
  - vllm/v1/core/sched/scheduler.py — V1 调度器核心：request queue、preemption、chunked prefill
  - vllm/v1/core/sched/async_scheduler.py — 异步调度器
  - vllm/v1/core/sched/output.py — 调度输出结构

2.3 KV Cache 管理
  - vllm/v1/core/kv_cache_manager.py — KV Cache 分配与回收
  - vllm/v1/core/kv_cache_utils.py — KV Cache 工具函数
  - vllm/v1/core/block_pool.py — Block 池管理
  - vllm/v1/core/single_type_kv_cache_manager.py — 单一类型 KV Cache 管理
  - docs/design/paged_attention.md — 理解 PagedAttention 的核心思想（类似 OS 虚拟内存分页）

2.4 Worker（执行器）
  - vllm/v1/worker/gpu_worker.py — GPU Worker 入口
  - vllm/v1/worker/gpu_model_runner.py — 模型执行的核心（准备输入 tensor、调用 model.forward、CUDA graph）
  - vllm/v1/executor/multiproc_executor.py — 多进程执行器（spawn worker 子进程）

2.5 请求模型与 I/O
  - vllm/v1/request.py — 请求生命周期
  - vllm/v1/engine/input_processor.py — 输入处理（tokenization）
  - vllm/v1/engine/output_processor.py — 输出处理（detokenization、streaming）
  - vllm/sampling_params.py — 采样参数定义

================================================================================
第三阶段：模型加载与执行（3-5 天）
================================================================================

目标：理解 vLLM 如何加载模型权重、执行 forward pass，以及分布式并行策略。

3.1 模型加载流程
  - vllm/model_executor/model_loader/ — 模型加载器（从 HuggingFace / 本地 / S3 等加载权重）
  - vllm/config/load.py — 加载配置
  - vllm/config/model.py — 模型配置（架构、dtype、max_model_len 等）
  - docs/design/huggingface_integration.md — HF 到 vLLM 模型的映射机制

3.2 模型定义体系
  - vllm/model_executor/models/ — 浏览 registry.py 理解模型注册机制，选取一个简单模型（如 opt.py、gpt2.py）作为入口
  - vllm/model_executor/layers/ — 基础层实现（linear、attention、norm、rotary_embedding 等）

3.3 分布式并行
  - vllm/distributed/ — 通信原语（all-reduce、broadcast 等）
  - vllm/config/parallel.py — 理解 TP（Tensor Parallelism）、PP（Pipeline Parallelism）、DP（Data Parallelism）的配置
  - vllm/distributed/parallel_state.py — 进程组管理与 NCCL 通信
  - docs/serving/parallelism_scaling.md — 并行策略文档

3.4 自定义算子与内核
  - vllm/model_executor/custom_op.py — 自定义算子注册
  - vllm/model_executor/kernels/ — Python 侧内核封装
  - csrc/ — C++/CUDA 实现的内核（可以只看注释了解功能）

================================================================================
第四阶段：Attention 与 KV Cache 优化（2-3 天）
================================================================================

目标：理解 vLLM 中多种 attention 后端的架构及其优化技术。

4.1 Attention 后端体系
  - docs/design/attention_backends.md — 理解各 attention 后端的架构
  - vllm/v1/attention/backends/registry.py — 后端注册与选择
  - 阅读 1-2 个核心后端实现：
    - vllm/v1/attention/backends/flash_attn.py — FlashAttention 后端（最常用）
    - vllm/v1/attention/backends/flashinfer.py — FlashInfer 后端
    - vllm/v1/attention/backends/mla/ — MLA（Multi-head Latent Attention，DeepSeek V2/V3 使用）

4.2 Attention Ops
  - vllm/v1/attention/ops/paged_attn.py — PagedAttention 实现
  - vllm/v1/attention/ops/chunked_prefill_paged_decode.py — Chunked Prefill 与 Paged Decode
  - vllm/v1/attention/ops/prefix_prefill.py — Prefix Prefill

4.3 Prefix Caching
  - docs/design/prefix_caching.md — Prefix Caching 设计
  - docs/features/automatic_prefix_caching.md — APC 使用指南

================================================================================
第五阶段：采样、结构化输出与推测解码（2-3 天）
================================================================================

目标：理解解码阶段的采样策略和加速技术。

5.1 采样
  - vllm/v1/sample/sampler.py — 采样器核心
  - vllm/v1/sample/rejection_sampler.py — 拒绝采样（用于 speculative decoding）
  - vllm/v1/sample/logits_processor/ — Logits 处理器
  - vllm/logits_process.py — 自定义 logits 处理器接口

5.2 结构化输出
  - vllm/v1/structured_output/ — 结构化输出后端（xgrammar、outlines、guidance 等）
  - docs/features/structured_outputs.md — 结构化输出文档

5.3 推测解码 (Speculative Decoding)
  - vllm/v1/spec_decode/ — 推测解码框架
    - eagle.py / medusa.py — EAGLE / Medusa 推测解码
    - ngram_proposer.py — N-gram 推测解码
    - draft_model.py — 草稿模型
  - docs/features/speculative_decoding/ — 推测解码文档

================================================================================
第六阶段：高级特性（2-3 天）
================================================================================

6.1 LoRA 适配器
  - vllm/lora/ — LoRA 实现
  - docs/features/lora.md — LoRA 文档

6.2 多模态
  - vllm/multimodal/ — 多模态处理（图像、音频、视频）

  - docs/design/mm_processing.md — 多模态处理设计文档

6.3 KV Cache Offload

6.4 分离式推理 (Disaggregated Prefill/Decode)
  - docs/features/disagg_prefill.md — 分离式 Prefill 与 Decode

6.5 Plugins 插件系统
  - vllm/plugins/ — 插件系统
  - docs/design/plugin_system.md — 插件设计文档

================================================================================
推荐的工具与调试方法
================================================================================

1. 日志追踪：设置 VLLM_LOGGING_LEVEL=DEBUG 查看详细日志
2. Python 调试：在关键路径加 breakpoint() 或使用 pdb 单步调试
3. 可视化：用 nsys 或 perf 分析 CUDA kernel 性能
4. 测试驱动理解：阅读 tests/v1/core/test_scheduler.py 等测试文件，理解各组件的预期行为
5. Git 历史：对于复杂模块（如 PagedAttention），用 git log --follow 跟踪演进历史
